# Sudoku Puzzle

## Repository 
https://github.com/paulahernan/pfda-final-project.git 

## Description
I will create a program which randomizes sudoku puzzles that can be completed with user input. This applies to media and digital arts since puzzles are a good strategy to make interactive games more interesting. 

## Features
1- Randomized Sudoku Puzzles (3 for each difficulty)
    - This will be executed through coding each puzzle within the program which will then be output randomly through the Random library depending on the difficulty option chosen.
2- Game interface
    - This will be executed through Pygame library's display, fonts, event types and drawing options. I will also use grids for how the sudoku puzzles numbers will display.
3- Options- Easy, medium, hard
    - This will be executed through event loops which will execute the sudoku puzzle depending on the difficulty chosen.
4- Reseting and clearing game 
    - This will be executed through event loops and Pygame keydown event types which will execute the event.

## Challenges
- I would need to make more research into Pygame and how to make more visually complex programs with the tools provided by that library.
- I will need to learn more about grids and how to incorporate them within Pygame and how it will be shown visually within the program.
- Finally, I will also need to learn how exactly I could make the program check and understand how the Sudoku puzzle should work and be correct or incorrect.

## Outcomes 
Ideal Outcome:
    - The ideal outcome is a fully functional program which lets you fist choose the diffciulty of the puzzle- that then leads you to a randomized puzzle out of three with the chosen difficulty that can be completed with user input as well as reset and cleared.
Minimal Viable Outcome:
    - The minimal viable outcome is a program that can display at least one sudoku puzzle that can be completed with user input.
